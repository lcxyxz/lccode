# Fix: Terminal Scrolling Performance

## Problem

Ink rewrites the entire screen on every state change. Long conversations cause:
- Mouse scroll lag (Ink rewrites fight terminal scrollback)
- All sections rendered without virtualization → linear rendering cost
- Streaming responses trigger高频 re-renders

## Solution: Static + Dynamic Split Rendering

Use Ink's `<Static>` component for completed sections (printed once, never re-rendered). Only the latest response stays dynamic (for streaming updates). Terminal native scrollback works naturally for static content.

## Files to Modify

### 1. `src/frontend/hooks/useOutput.ts`

**Changes:**
- Add `staticSections` state to hold completed sections (printed via `<Static>`)
- Keep `activeSections` for the latest section being updated
- When a new section is added, the previous active section moves to static
- `trimSections` trims from the static buffer (oldest first)
- Remove unused `hideLatestRef`

**New state shape:**
```ts
const [staticSections, setStaticSections] = useState<OutputSection[]>([])
const [activeSection, setActiveSection] = useState<OutputSection | null>(null)
```

**New return value:**
```ts
return {
  staticSections,   // for <Static> component
  activeSection,    // latest section (dynamic)
  addInput,
  addCommand,
  addResponse,
  clearSections,
  trimSections,
}
```

**Logic for `addSection`:**
- If there's an existing `activeSection`, move it to `staticSections`
- Set the new section as `activeSection`
- This ensures only 1 section is ever dynamic

### 2. `src/app.tsx`

**Changes:**
- Import `Static` from `ink`
- Destructure `staticSections` and `activeSection` from `useTerminal`
- Render `<Static>` with `staticSections`, render `activeSection` normally

```tsx
<Box flexDirection="column">
  <Header llmStatus={llmStatus} />
  <Static items={staticSections}>
    {(item) => <OutputSection key={item.id} section={item} />}
  </Static>
  {activeSection && <OutputSection section={activeSection} />}
  {/* ... rest of components */}
</Box>
```

### 3. `src/frontend/useTerminal.ts`

**Changes:**
- Destructure `staticSections` and `activeSection` instead of `sections: visibleSections`
- Merge them for `trimSections` call (trim from static only)
- Pass merged `sections` (static + active) to components that need the full list

**New destructuring:**
```ts
const {
  staticSections, activeSection,
  addInput, addCommand, addResponse,
  clearSections, trimSections,
} = useOutput(availableHeight)
```

**For `trimSections`:** Call with `staticSections.length` to trim oldest static items.

### 4. `src/frontend/components/OutputLines.tsx`

**Changes:**
- Simplify to only render the active section (no longer maps over full array)
- Or remove entirely if `app.tsx` handles rendering directly

## How It Fixes the Problem

| Before | After |
|--------|-------|
| Ink rewrites ALL sections on every render | Ink only rewrites 1 section (the active one) |
| Terminal scrollback disrupted constantly | Static content stays stable in scrollback |
| Rendering cost O(n) per update | Rendering cost O(1) per update |
| Mouse scroll fights Ink re-render | Terminal scrollback works naturally |
